How it works

  • Frame the question: yes/no for humans, or a bounded set for agents.
  • Decision API returns the verdict with rationale and machine-readable fields.
  • Relevant notary executes (refund, cancel, return, trial) via MCP or REST.
  • Audit trail (optional) stores payload, response, assertions, and diffs.

When to use it

  • Policy-bound flows that must be consistent (refund/cancel/return/trial).
  • Agent routing: pick the correct notary before acting.
  • Product UI that needs a crisp yes/no without hedging.
  • QA and regression checks with reusable suites.

For agents

Agents call the same Decision API your users see. Verdicts map to notary actions with deterministic payloads. No “maybe,” no vibe-based prompts.

For humans

A clean yes/no interface without the old “magic 8 ball” gimmickry. When there’s no meaningful difference, we tell you it’s a tie so you can move on.
